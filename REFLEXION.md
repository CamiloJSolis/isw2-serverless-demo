## 1. ¿Qué tipo de error evita el CI?
Evita el error de meter código nuevo sin que pase pruebas automáticas antes de llegar a la rama principal. Detecta rápido problemas de integración, builds rotos y regresiones.

## 2. ¿Qué tipo de error no evita?
Errores de seguridad, mala arquitectura, configuraciones mal puestas, problemas operativos o rendimiento en producción real. Básicamente todo lo que no se cubre con pruebas automáticas básicas.

## 3. ¿Qué pasaría si un equipo ignora el CI?
Se arma un desastre: provoca integraciones tardías, mayor número de errores y despliegues más riesgosos. Además, se pierde la capacidad de realizar entregas continuas y confiables basadas en pruebas automatizadas, que es el corazón de CI/CD.
